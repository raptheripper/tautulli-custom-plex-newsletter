# Tautulli Custom Plex Newsletter
Enhanced responsive Tautulli newsletter featuring poster & metadata caching, library grouping, rating badges, Cloudinary support, self-hosted asset hosting, Cloudflare custom domain integration, animated counters and optional self-hosted browser view.

---

## Features

### Enhanced Newsletter Layout

- Responsive multi-column design
- Library-based grouping
- Custom Plex branding/logo support
- Modern movie and TV show cards
- Age rating badges
- Genre badges
- TMDb ratings for TV shows
- Rotten Tomatoes ratings for movies
- Filter system based on Plex labels
- Animated counters (Apple Mail / supported clients)
- Static fallback counters for Gmail and Outlook
- Hidden email preview text optimization

## Poster & Metadata Cache

The included plex_update_cache.py script creates a local metadata cache containing:

- Plex Watch URLs
- Poster URLs
- Labels
- Library names

This cache is used by the newsletter template to:

- Generate Plex Watch links
- Load cached posters
- Filter content using Plex labels
- Group content by library

# Installation

## Directory Structure

    docker/
    ├─ plex-cache/
    │  ├─ public/
    │  │  ├─ cache/
    │  │  │  ├─ posters/
    │  │  │  ├─ plex_cache.json
    │  ├─ python/
    │  │  ├─ plex_update_cache.py
    │  ├─ nginx/
    │  │  ├─ default.conf
    │  ├─ .env
    │  ├─ requirements.txt
    │  ├─ compose.yml

Create the above directory structure and place all of the included files in their respective directories as shown above. 

***NOTE:** The file plex_cache.json is not included, this file gets generated by running the plex_update_cache.py script.*

## Docker Compose

***Example:***
   

     services:
        
          plex-cache:
            image: python:3.11-slim
            container_name: plex-cache
        
            network_mode: host
            working_dir: /python
        
            env_file:
              - .env
        
            volumes:
              - ./python:/python
              - ./public/cache:/cache
              - ./requirements.txt:/requirements.txt
        
            command: >
              sh -c "
              pip install --no-cache-dir -r /requirements.txt &&
              tail -f /dev/null
              "
        
            restart: unless-stopped
        
        
          poster-cache:
            image: nginx:alpine
            container_name: poster-cache
        
            ports:
              - "8095:80"
        
            volumes:
              - ./public:/usr/share/nginx/html:ro
              - ./newsletters:/newsletters:ro
              - ./nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
        
            restart: unless-stopped 

The included docker-compose.yml example needs to be edited before creating the docker containers. The example file includes two containers:

### plex-cache container
This container installs:

 - Python 3.11-slim
 - pip
 - plexapi 4.16.0
 - Pillow 10.0.0
 - requests 2.31.0
 - cloudinary 1.41.0

**Volumes of the plex-cache container**
Replace all . (periods) with the actual path to your docker folder.

***Example:***

Replace: 

      - ./python:/python
      - ./fonts:/fonts
      - ./public/cache:/cache
      - ./requirements.txt:/requirements.txt

With:

      - /volume1/docker/plex-cache/python:/python
      - /volume1/docker/plex-cache/fonts:/fonts
      - /volume1/docker/plex-cache/public/cache:/cache
      - /volume1/docker/plex-cache/requirements.txt:/requirements.txt
---
### poster-cache container
If Cloudinary is used for poster hosting and all other assets are hosted elsewhere, the nginx container is optional. However, without the nginx container you will lose the self-hosted browser view functionality unless newsletters are hosted using another web server.

***NOTE:** If you disable the nginx container, remove or comment out the
`"View this email in your browser"` section in recently_added.html, or replace it with your own newsletter hosting solution.*

This container installs:

 - nginx:alpine

**Volumes of the poster-cache container**
Replace all . (periods) with the actual path to your docker folder.

***Example:***

Replace: 

      - ./public:/usr/share/nginx/html:ro
      - ./newsletters:/newsletters:ro
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf:ro

With:

      - /volume1/docker/plex-cache/public:/usr/share/nginx/html:ro
      - /volume1/docker/tautulli/config/newsletters:/newsletters:ro
      - /volume1/docker/plex-cache/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro

## Environment Variables

The included docker-compose.yml example uses:

    env_file: 
      -.env

1. Update the values for your environment in the included example.env file
 2. Copy the file to plex-cache root folder
 3. rename example.env to .env

**Required variables:**

### Plex

    PLEX_URL=http://your-ip:32400 
    PLEX_DOWNLOAD_URL=http://your-ip:32400
    PLEX_TOKEN=YOUR_PLEX_TOKEN

---

### Local Poster Hosting

    POSTER_PROVIDER=local
    POSTER_PUBLIC_URL=https://assets.example.com/cache/posters

---

    CLOUDINARY_CLOUD_NAME=xxxxxxxx
    CLOUDINARY_API_KEY=xxxxxxxx

    CLOUDINARY_FOLDER=plex-posters

# Self-Hosted Assets

- Plex logo

    https://example.com/logo.png https://example.com/ratings/...

- Nginx
- Cloudflare
- Cloudinary
- GitHub Pages
- Any public HTTPS web server

# Cloudflare Custom Domain Setup

A common setup is:

     assets.yourdomain.com

pointing to your Nginx container at `http://your-ip:8095 `.

Example:

     https://assets.example.com/cache/posters/12345.webp

Benefits:

- HTTPS
- CDN caching
- Stable poster URLs
- Faster newsletter loading

# Automated Nightly Cache Refresh

## OPTION 1. Synology DSM Task Scheduler

The recommended method on Synology DSM is to run the cache update script using Task Scheduler.

### Create a Scheduled Task

Open:

    Control Panel
    → Task Scheduler
    → Create
    → Scheduled Task
    → User-defined script

Configure:

    Task: Plex Cache Refresh
    User: root
    Schedule: Daily
    Time: 03:00

### User-defined Script

    docker exec plex-cache python /python/plex_update_cache.py

### Recommended Schedule

Run the task once every night before newsletters are generated.

 Example:

    03:00 - Refresh cache
    06:00 - Send newsletter

This ensures:

- New posters are cached
- Metadata is updated
- Plex Watch URLs remain current
- Cloudinary uploads are refreshed (if enabled)

### Viewing Logs

Task execution logs can be viewed in:

    Control Panel
    → Task Scheduler
    → Task
    → View Result

If the script runs successfully, you should see output similar to:

    [POSTER PROVIDER] local
    Loaded cache: 2450 items
    [POSTER] Cached: 12345.webp
    Done. Total cache: 2450 items
---

## OPTION 2: Cron job

    cron 0 3 * * * docker exec plex-cache python /python/plex_update_cache.py

This keeps:

- Posters updated
- Plex Watch URLs refreshed
- Metadata cache current

# Tautulli Configuration

The included custom newsletter template file:

    recently_added.html

must be imported into:

    docker/tautulli/config/custom-newsletter-templates

If this directory does not exist, create it.

Next, log in to Tautulli, go to:

    Settings  → Notifications & Newsletters → NEWSLETTERS → Custom Newsletter Templates Folder

Use the Browse button and browse to 

    docker/tautulli/config/custom-newsletter-templates

Click on Save.

***NOTE 1:** The python script plex_update_cache.py should be executed before newsletters are generated.* 

***NOTE 2:** Restart Tautulli after changing the custom template folder path if the templates do not appear immediately.*

# Troubleshooting

## Refreshing a cached poster

If a poster has been changed in Plex but the old cached version is still being used in the newsletter:

1. Find the item's Rating Key in Plex *(More → Get Info → View XML)*.
2. Delete the corresponding poster file:

       /cache/posters/<rating_key>.webp

3. Run the cache update script again.

The new poster will be downloaded to the cache.
